GIT COMMANDS AND RULES

Git Commands

  • $ git remote -v Ver a qué repo estamos conectados
  • $ git branch Ver en qué rama estamos trabajando
  • $ git checkout -b {branch} Actualizar HEAD, crear una nueva rama y cambiar a ella
  • $ git checkout {branch} Cambiar a una rama ya creada
  • $ git push {desde dónde} {a qué rama} Enviar los cambios de los archivos seleccionados desde la ubicación de trabajo a la rama especificada
  • $ git switch - Cambiar a la rama en la que nos encontrábamos anteriormente
  • $ git add . Stage-ear todos los archivos de la rama actual
  • $ git commit -m “{}” Hacer un commit de los cambios stage-eados con el mensaje especificado
  • $ gut pull {desde dónde} {rama} Traer los cambios desde la rama especificada a la ubicación de trabajo especificada
  • $ git status Mostrar todos los archivos en los que se han realizado cambios
  • $ git diff Mostrar los cambios realizados en los archivos
  • $ git config pull.rebase false Abrir el editor para elegir qué cambios persisten en los archivos conflictivos en el siguiente git pull

Git Golden Rules

  • Lo que no te toca no se toca
  • Nunca se trabaja en main
  • Nunca se trabaja en dev
  • Siempre se hace el push a la rama de trabajo
  • Las ramas de trabajo tienen que tener nombres descriptivos
  • Los mensajes del commit deben ser descriptivos
  • Antes de un push va un pull
  • Antes de comenzar a trabajar en tu rama, hacer un pull de dev
  • Cada vez que alguien modifique develop, hacer un pull
  • Al crear una rama, hacer un push de esa rama para crearla en github
